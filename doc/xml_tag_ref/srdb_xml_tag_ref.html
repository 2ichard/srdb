<equipment>

The <equipment> tag identifies an item to be included in a list of required tools and equipment. Equipment lists are associated with sections; therefore, <equipment> tags must be within <section></section> tags. A sepearate equpment list will be generated for each section from the <equipment> tags within that section.

Items in the list are numbered automatically according to the order of the <equipment> tags in the XML document. Each equipment list starts from 1.

If there are no <equipment> tags in a section then no equipment list will appear in that section.
Attributes
  None.
Example

An equipment list containing 5 items:

<section title="Inspect Segment>
<intro>In this section, a segment will be inspected.</intro>
<equipment>Standoff posts (30 items, 10 for each whiffletree).</equipment>
<equipment>0.001" shim stock.</equipment>
<equipment>Hot glue removal tool.</equipment>
<equipment>Methanol.</equipment>
<equipment>Cleaning tissue.</equipment>
<step>First step.</step>
<step>Sectond step.</step>
</section>

<figure>

The <figure> tag describes a figure but not the location at which it appears in the procedure.

Figures are described by a figure number, image file and an optional caption.

In contexts where a thumbnail version of the image is required, it is assumed to have the same file name as the full size image but located in the thumbnails directory which is immediately subordinate to the images directory.

Figures can appear in the following places:

  • In the procedure introduction (see <intro> contained inside a <procedure>).



  • In a section introduction (see <intro> contained inside a <section>.


  • In a step (see <step>).


In addition, links to figures can be included in the body of <intro>, <step> and <substep> tags.

No matter how many times a figure, its thumbnail or links to the figure appear in the document, only one <figure> tag is needed per figure. Image files can be of type JPEG, PNG or GIF. To include a link to a figure in body text, refer to the figure as Figure num, e.g. Figure 2.

The body of this tag, if included, represents the figure caption (see the example).

See also the figure attributes of the <intro>, <step> and <substep> tags.
Attributes
num The number of this figure, typically a number, e.g. num="1", but can be any text string, e.g. num="14-a". The figure number is used to label figures in the document and to reference figures in <intro> and <step> tags and in links to figures.
image The name of the image file for this figure. Image files are stored in a standard location so image file names are not proceeded by a path, e.g. image="WT_transportation_restraints.jpg".
Example

A figure with a caption but no thumbnail which therefore can only be used in the procedure introduction or referred to in textual links:

<figure num="1" image="whiffletree_color.png">
Whiffletree color layout for two segment types.
</figure>

A figure with a thumbnail and a caption which therefore can be used anywhere:

<figure num="2" image="WT_transportation_restraints.jpg" thumbnail="WT_transportation_restraints_120.jpg">
ASCII Import GUI: Importing pointing file for whiffletree.
</figure>

A reference to a figure in the body of a <step>:

<step>
Place the laser tracker on the kinematic adapter in the segment central hole (Position B in Figure 1).
</step>

<general>

The <general> tag identifes a general procedure to be included as a reference in a segment procedure. The num attribute specifies the procedure number to be referenced (e.g. SRP601) and is required.

This tag contains no text between the opening and closing tags. The procedure title used in the list of general procedures (below) are obtained dynamically from the database.

A list of referenced general procedures appears toward the top of the segment procedure:

Each item in the list is a link to the general procedure.

In addition, references to the general procedure in the present procedure (e.g. SRP601) are replaced with links to the general procedure.

Any number of <general> tags may appear in the document but they must be within the main <procedure> tag and outside any <section> or <intro> tags.
Attributes
num The number of the general procedure, e.g. SRP601
Example

<general num="SRP601"></general>
<general num="SRP604"></general>

<html>
The <html> tag delineates a section of body text that should be interpreted as literal HTML. Within an <HTML> tag body, standard HTML tags are used in their normal sense and are passed directly to the browser without further processing by the SRDB system. <HTML> elements can be used in the body of <intro>, <section>, <step> and <substep> tags.
Attributes
  None.
Example
Inserts a table:

<html>
<table align="center">
<tr><td>Segment Types 1, 3, 5, 6</td><td>&nbsp;</td><td>Red1, Blue1, Green1</td></tr>
<tr><td>Segment Types 2 and 4</td><td>&nbsp;</td><td>Red2, Blue2, Green2</td></tr>
</table>
</html>
<intro>

An <intro> element represents a procedure introduction if it is in the body of a <procedure> element or a section introduction if it is in the the body of a <section> element. <intro> elements are used to provide introductory or background information related to the procedure as a whole or to section of a procedure. <intro> elements consist of text and an optional figure. The text is body of the tag and the figure is specified as an attribute.

A figure in a procedure introduction is included in-line as a full-size image. A figure in a section introduction is included as a thumbnail which links to the full-size image displayed in a new tab or window.
Attributes
figure In a procedure introduction the figure is displayed full size at the end of the text, in a section introduction the figure is displayed as a thumbnail to the right of the text and links to the full-size image.
Example
<intro figure="2">
The segment is delivered from the summit with the radial post removed and all three whiffletrees restrained. Refer to Figure 2 which shows a whiffletree with mounted restraints.
</intro>
<p>
The <p> element represent a paragraph break and may be included in the body of an <intro>, <step> or <substep> tag. Note that <p> is the same as <html><p></html>.
Attributes
  None.
Example
<intro>
This is the first paragraph.
<p>
And this is the second paragraph.
</intro>
<procedure>
The <procedure> element represent an SRDB procedure. There is only one <procedure> element in a document.
Attributes
id Identifier for this procedure, e.g. SRB1234.
title Title of procedure.
version Version of procedure.
type Type, segment or general.
Example
<srdbxml>

<procedure num="SRDB0000" version="1.0" title="Whiffletree Measurement with Laser Tracker">

<figure num="1" image="whiffletree_color.png">Whiffletree color layout for two segment types.</figure>

<equipment>0.5" SMR brackets (42 items, 14 for each whiffletree).</equipment>

<intro figure="1">Donec at ultrices velit.</intro>

<section title="Setup Laser Tracker" signoff="true">
...
body of section
...

</section>
</procedure>
</srdbxml>
<section>
Represents a section in a procedure. A <section> may contain an optional <intro> and one or more <step> elements. Sections are automatically numbered according to the order in which they appear in the XML document.
Attributes
title The title of this section.
signoff If true or section (e.g. signoff="true" or signoff="true") then a section sign-off widget is included at the end of this section, if false or omitted then no sign-off widget is included. No other sign-off type is allowed in the section tag. See Sign-Offs for more information.
Example
<section title="Setup Laser Tracker" signoff="true">

<intro>
This is the introduction to section 1 which does not have a figure.
</intro>

<step>
Laser Tracker positioned on the kinematic adapter in the segment central hole (Position B in Figure 2).
</step>

<step>
Vivamus pulvinar orci ligula, vel laoreet neque convallis lacinia.
</step>

</section>

Identifies the document as SRDB XML procedure description.
Attributes
  None.
Example
<srdbxml>
<procedure num="SRDB0000" version="1.0" title="Whiffletree Measurement with Laser Tracker">
...
body of procedure
...
</procedure>
</srdbxml>
<step>

A step in a procedure section. Steps may only appear in sections. Steps may but need not include sub-steps. The body of this element contains the instructions for this step. Steps are numbered automatically according to the order in which they appear in the XML document.

If a step has substeps the steps may be made collapsable by setting the collapsable attribute to true. The figures below show a step with substeps expanded and then collapsed.



If collapsable is set to false then the substeps are shown expanded, the collapse control is not available and the substeps cannot be collapsed.

If collapsible is set to true then the initial state of the substeps is determined by the collapsed attribute. If collapsed is set to true then the substeps are initially show collapsed and the user can expand them by clicking on the Expand Substeps control.

Attributes
signoff If true (e.g. signoff="true") then a step sign-off widget is included at the end of this step, if false or omitted then no step sign-off widget is included. If signoff is set to a number, e.g. signoff="3", then an n-part signoff is included.
figure Number of figure to be included as a thumbnail which links to a full-size image in a separate window or tab.
collapsed If true then substeps, if any, are initially shown collapsed, i.e. not visible. The default value is false, i.e. substeps, if any are shown.
collapsable If true, and if there are substeps, then controls are made available for this step to expand and collapse the substeps. If false then substep expand and collapse controls are not visible and the initial collaps state cannot be changed.
Example

<step figure="4" signoff="3">
Import three ASCII pointing files according to segment type 1 or 2 as shown in the screenshot (Figure 4).
</step>

In this example, there are two substeps, the substeps are shown and the user cannot hide them:

<step collapsed="false" collapsable="false">
This is the text that describes the step.
<substep>This is the first substep></substep>
<substep>This is the second substep></substep>
</step>

In this example, there are two substeps, the substeps are initially collapsed (hidden) but controls are made available to expand (show) the substeps:

<step collapsed="true" collapsable="true">
This is the text that describes the step.
<substep>This is the first substep></substep>
<substep>This is the second substep></substep>
</step>
<substep>
A substep within a step. Substeps are numbered automatically according to the order in which they appear in the XML document.
Attributes
  None.
Example
<step>
Aliquam nunc nibh, iaculis et purus convallis, pretium egestas odio.

<substep>
Pellentesque quis quam semper, eleifend arcu at, condimentum tortor.
</substep>

<substep>
Cum sociis natoque penatibus et magnis dis parturient montes, nascetur ridiculus mus.
</substep>

</step>

Sign-offs are checkpoints where the technician certifies that a step or section is complete and in some cases records information like the identity of a fixture used to complete a certain step or the lot number of an adhesive used in a step. When a sign-off is signed-off, information associated with the sign-off is stored in the database. The state and data associated with a sign-off for the currently selected segment are shown on the procedure pages.

There are two predefined types of sign-offs--step and section signoffs--and an unlimited number of custom sign-offs. Step sign-offs are used to record completion of a step when no special data is required to be recorded with the step. Step sign-offs appear at the end of a step. Before the step is signed-off, it appears as in the figure below, with a greenish top row and a reddish bottom row.

Sign-Offs

After the technician has signed-off on the step, the sign-off appears as in the next figure with greenish top and bottom rows.

To include a step sign-off in a step add the signoff attirbute to the step tag, e.g. <step signoff="true"> or <step signoff="step">.

Section sign-offs are similar to step sign-offs but appear at the end of a section. Use the signoff attribute in the section tag to include a section sign-off, e.g. <section signoff="true"> or <section signoff="section">.

Custom sign-offs can be defined on the Sign-Off Definition page.

Type is the identifier which will be used in the XML tag, e.g. <step signoff="inner_fixture">, and should contain no spaces or special characters. Title is the name of the sign-off that will appear in the upper left corner of the signoff widget on the procedure page. Label will appear just before the drop down menu of choices. Choices are the menu items from which the technician must select a value when signing-off on the step and is a comma-separated list of strings.
